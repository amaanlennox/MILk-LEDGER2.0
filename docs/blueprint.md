# **App Name**: Milk Diary

## Core Features:

- Customer Management: Admin function to add, edit, or delete customer profiles, including their name, optional default milk quantity, and default rate per litre.
- Customer Home Screen: Displays a list of all registered customers with easy navigation buttons to perform daily entries or view monthly summaries for a selected customer.
- Daily Milk Entry: Allows selection of a customer, displays their default milk rate, and provides an input field for milk quantity. Automatically calculates the total amount (quantity × rate) and saves the entry with the current date.
- Local Data Storage: Persistently stores all customer information and daily milk entries directly on the device, ensuring the app is fully functional offline without requiring user login.
- Monthly Summary: Provides a clear, list-based view of a selected customer's daily entries for the current month, along with aggregated totals for milk quantity and the overall bill amount.
- Rate Adjustment: Enables administrators to easily update or set the default milk rate per litre for individual customers.
- Multilingual Labels: Basic support for switching user interface labels and text between Hindi and English.

## Style Guidelines:

- Primary Color: A calming, muted sky blue (#6BA3EC) to signify cleanliness and clarity, contrasting effectively with the background for active elements.
- Background Color: A very light, almost white blue-gray (#EEF3F7), providing a spacious and easy-to-read canvas for content, especially for older users.
- Accent Color: A refreshing turquoise (#33C2C2), used sparingly for important actions or highlights to provide a clear focal point.
- Body and Headline Font: 'Inter' (sans-serif), chosen for its excellent legibility, modern appearance, and neutrality, ensuring readability across different content types and for all users, including those with visual impairments.
- Icons will be simple, solid, and easily recognizable representations of common actions like 'Add Customer,' 'Save Entry,' or 'View Summary,' designed to be intuitive for all users.
- The layout will be spacious and clean, with ample white space around elements. Key interactive elements, particularly buttons and input fields, will be large to ensure easy interaction for older users and reduce accidental taps.
- Animations will be minimal and functional, such as subtle transitions between screens or discrete visual feedback upon button presses, prioritizing speed and clarity over elaborate effects to maintain a simple user experience.
